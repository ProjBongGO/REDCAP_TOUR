# AI 작업 규칙

## 1. 기본 원칙
- **명확한 목표**: 작업 전에 목표를 명확히 정의하고, 결과물이 어떻게 사용될지 고려합니다.
- **분산 처리**: 복잡한 작업은 단계별로 나누어 처리하며, 각 단계마다 검토를 진행합니다.
- **문서화**: 작업 과정과 결과는 적절히 문서화하여 추후 참고할 수 있도록 합니다.

## 2. 파일 관리
- **명명 규칙**: 파일명은 의미가 명확하도록 작성하며, 한글과 영어를 혼용할 때는 주의합니다.
- **버전 관리**: 중요한 파일은 버전을 관리하고, 변경 사항은 커밋 메시지로 기록합니다.
- **백업**: 작업 전후로 중요한 파일은 백업을 진행합니다.

## 3. 코드 작성
- **가독성**: 코드는 가독성이 높게 작성하며, 주석을 적절히 사용합니다.
- **재사용성**: 공통으로 사용되는 코드는 모듈화하여 재사용할 수 있도록 합니다.
- **테스트**: 코드 작성 후 테스트를 진행하여 오류를 발견하고 수정합니다.

## 4. 데이터 처리
- **유효성 검사**: 입력 데이터는 유효성 검사를 통해 오류를 방지합니다.
- **보안**: 민감한 데이터는 암호화하거나 안전한 방법으로 관리합니다.
- **일관성**: 데이터의 형식과 구조는 일관성 있게 유지합니다.

## 5. 문제 해결
- **분석**: 문제를 분석하여 근본 원인을 찾습니다.
- **대안**: 여러 대안을 고려하여 최적의 해결 방법을 선택합니다.
- **예외 처리**: 예외 상황은 적절히 처리하여 시스템의 안정성을 높입니다.

## 6. 협업
- **소통**: 팀 내에서 작업 내용을 적절히 소통합니다.
- **규칙 준수**: 팀에서 정한 규칙을 준수합니다.
- **피드백**: 피드백을 받아 작업을 개선합니다.

## 7. 성능 최적화
- **코드 최적화**: 불필요한 코드는 제거하고, 성능을 높이기 위한 최적화를 진행합니다.
- **리소스 관리**: 리소스 사용을 모니터링하고, 필요한 경우 조정합니다.
- **캐싱**: 반복적으로 사용되는 데이터는 캐싱하여 성능을 향상시킵니다.

## 8. 보안
- **인증**: 시스템 접근은 적절한 인증 방법을 사용합니다.
- **인가**: 사용자의 권한을 적절히 관리합니다.
- **로그**: 중요한 작업은 로그를 남겨 추적할 수 있도록 합니다.
